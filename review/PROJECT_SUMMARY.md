## アプリ概要

**アプリ名:** 食品商品管理クラウド（かんたん食品表示ラベルメーカー）
**GitHub:** https://github.com/keisuke1983/easy-food-label-maker
**現バージョン:** v20260710h（2026-07-10）

食品メーカー・小規模食品事業者が「食品表示ラベル作成」「商品マスター管理」「規格書作成」「AI食品表示チェック」を専門知識なしでワンストップに完結できる AI SaaS。

---

## ターゲット

| セグメント | 規模 |
|-----------|------|
| 年商〜3億円規模の食品メーカー | 主ターゲット |
| 農家直売所・産直EC事業者 | サブターゲット |
| 菓子製造業者・惣菜店・ベーカリー | サブターゲット |

**ペルソナ:** 食品表示法の専門知識はないが、社内でラベルを作らなければならない担当者。ExcelやWordで手作りしており、法律違反リスクと作業工数の両方に悩んでいる。

---

## コンセプト

> 「食品表示の専門家がいなくても、法律に準拠したラベルが10分で作れる」

- **AIがチェック** → 入力した原材料・アレルゲン・栄養成分を自動検証
- **テンプレート不要** → 入力するとリアルタイムでラベルプレビューが更新される
- **クラウド不要** → ブラウザのみ。インストール不要。オフライン動作（PWA対応）

---

## 主要機能

### 商品管理（実装済み）
- 商品マスター CRUD（削除undo / 複製undo付き）
- 完成度スコア自動計算（0〜100%）
- 賞味期限アラート（30日以内・期限切れバッジ）
- 変更履歴管理（差分表示付き）
- 複合フィルター（カテゴリ AND 完成度 AND ステータス）
- 保存済み検索条件ブックマーク
- 商品ソート（更新日 / 名前 / 完成度）

### 食品表示ラベル（実装済み）
- 食品表示法準拠のラベルリアルタイムプレビュー
- PDF印刷 / PNG画像エクスポート
- サイズプリセット（70×50mm〜A4シール）+ 自由サイズ
- バーコード（JAN）表示対応
- 複数商品一括印刷

### AI機能（実装済み）
- AI食品表示チェック（OpenAI gpt-4o-mini）
- AI商品説明文生成（楽天 / Amazon / Yahoo / SNS チャネル別）
- AIチャット相談（商品文脈付き）
- 写真からの商品登録（gpt-4o Vision API）
- 規格書PDFからの情報抽出

### 原材料管理（実装済み）
- 原材料入力（重量順・添加物後列 自動ソート）
- アレルゲン自動検出（28品目対応）
- 栄養成分自動計算（原材料DBから）

### 原価管理（実装済み）
- 直接原価入力 / 明細入力
- 原価率・粗利率・粗利額 自動計算
- 規格書への原価情報表示切替

### チーム・承認機能（実装済み）
- チームメンバー管理（管理者 / 編集者 / 確認者）
- 承認ワークフロー（申請 → 承認 / 差し戻し）
- 承認ステータスバッジ（商品カード・サイドバー）

### データ管理（実装済み）
- CSV エクスポート/インポート（日本語ヘッダー自動変換）
- JSON フルバックアップ（画像あり / なし 選択式）
- PWA対応（Service Worker / manifest.json）

## 技術構成

| 項目 | 技術 |
|------|------|
| フロントエンド | Vanilla JavaScript（フレームワークなし） |
| スタイル | カスタム CSS（CSS変数ベーステーマ） |
| ストレージ | localStorage（商品データ）/ sessionStorage（APIキー） |
| AI | OpenAI API（gpt-4o / gpt-4o-mini）|
| ビルドツール | なし（CDNや依存ライブラリ不使用） |
| ホスティング | 静的HTML（GitHub Pages等でデプロイ可能） |
| PWA | Service Worker + Web App Manifest |

**アーキテクチャの特徴:**
- グローバル状態 + `render()` が `root.innerHTML` を全書き換えする純粋SPA
- イベント委譲パターン（`setupDelegation()` 一度だけ + `bindDynamic()` 各レンダー後）
- スクリプトロード順: `db.js → constants.js → utils-core.js → state.js → delegation.js → app.js`
