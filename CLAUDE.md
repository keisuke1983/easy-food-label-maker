# FoodPilot Project Instructions

## 1. Project Overview

FoodPilotは食品メーカー向けAI商品管理クラウドです。

単なる食品表示ラベル作成ツールではなく、
「食品メーカーのAI商品管理責任者」となることを目指しています。

食品企業の商品開発、品質管理、規格管理、商品改善、変更管理をAIで支援し、
商品に関する情報を一元管理するSaaSです。

現在のPhase：
Phase1 MVP開発・販売準備段階

現在の主要機能：

- 食品表示ラベル作成
- 原材料管理
- アレルゲン管理
- 栄養成分計算
- 商品登録
- AIによる文章生成
- 商品情報管理

今後追加予定：

- AI商品カルテ
- 商品変更履歴（タイムライン）
- AI改善提案
- AI規格書作成
- OEM案件管理
- AI棚分析
- 商品データからの経営判断支援


---

# 2. Product Vision

FoodPilotの目的は、
食品メーカーの商品管理業務をAI化し、
担当者の経験や勘に依存した業務から、
データとAIによる意思決定へ変えることです。

FoodPilotは以下を実現します。

- 商品情報を資産化する
- 商品変更リスクをAIで発見する
- 商品改善案をAIが提案する
- 商品開発スピードを向上する
- 食品企業の業務効率と利益率を改善する

開発判断では必ず以下を優先してください。

1. 食品メーカーが継続利用する価値があるか
2. 月額課金できる価値があるか
3. データが蓄積され将来AI活用できる設計か
4. 競合との差別化につながるか


---

# 3. Target Users

主な対象ユーザー：

- 食品メーカー
- OEMメーカー
- 食品加工会社
- 商品開発担当者
- 品質管理担当者
- 商品企画担当者

主な課題：

- 商品情報がExcelや紙に分散している
- 規格書作成に時間がかかる
- 食品表示変更管理が大変
- 原材料変更時の影響確認が難しい
- 商品改善の判断材料が不足している

# 5. Development Rules

必ず守ること：

- 既存機能を壊さない
- 実装前に現在のコード構造を確認する
- 影響範囲を確認してから変更する
- 必要以上に大規模な変更をしない
- シンプルで保守しやすいコードを書く
- 重複コードを増やさない
- セキュリティを優先する
- UI/UXは食品企業担当者が迷わず使えることを重視する


変更するときは以下を考える：

- なぜ必要なのか
- ユーザー価値は何か
- SaaS価値につながるか
- 将来的な拡張性があるか

商品データ：

- 商品名
- 商品画像
- 商品カテゴリ
- 原材料
- 配合情報
- アレルゲン
- 栄養成分
- 内容量
- 賞味期限
- 保存方法
- 規格書
- 食品表示
- 商品履歴
- AI分析結果


商品データは将来的にAIが分析できる形で保存する。

単なる表示データではなく、
食品企業の商品資産として扱う。


---

# 7. AI Feature Development Rules

AI機能は単なる文章生成ではなく、
食品メーカーの意思決定支援を目的にする。

良いAI機能：

例：
- 「原材料変更により表示変更が必要です」
- 「この商品の原価率改善案があります」
- 「競合商品と比較して改善ポイントがあります」
- 「賞味期限変更による影響があります」

避けること：

- 汎用的な文章生成だけの機能
- 他サービスとの差別化にならないAI機能


AIはFoodPilotの最大の差別化要素として考える。

機能追加の優先順位：

最優先。

## Priority 2

重要。

- 業務効率改善
- UX改善

## Priority 3

後回し。

- 見た目だけの改善
- 利用頻度が低い機能


売上・継続率につながらない大型開発は避ける。

# 10. UI/UX Guidelines

FoodPilotの利用者は食品業界の担当者。

IT専門家ではありません。

UIでは：

- 初めてでも迷わない
- 入力作業を減らす
- AIが次の行動を提案する
- 重要情報がすぐ確認できる

ことを優先する。
