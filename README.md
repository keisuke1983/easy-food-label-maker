# 食品商品管理クラウド

> 食品メーカー・小規模食品事業者向け AI 搭載商品管理 SaaS

**「食品表示法に準拠したラベル作成・原材料管理・商品規格書作成を、専門知識なしで完結させる」**

---

## 概要

食品の商品ラベル作成・原材料管理・規格書作成・原価管理を一元管理できる Web アプリケーションです。OpenAI API と連携した AI 機能により、食品表示法チェック・商品説明文生成・専門家への相談を自動化します。

バックエンドなし・登録不要で即日利用開始できます。データはブラウザのローカルストレージに保存されます。

---

## 特徴

- **説明書なしで使える** — 初めての食品担当者でも迷わない直感的 UI
- **食品表示法準拠** — アレルゲン 28 品目自動検出・原材料重量順ソート・栄養成分自動計算
- **AI が作業を自動化** — 食品表示法チェック・商品説明文・AI 相談チャット（OpenAI API 連携）
- **キーボード完全操作** — `/` 検索 / `N` 新規 / `Ctrl+S` 保存 / `Esc` キャンセル
- **安全なデータ管理** — 削除 undo / JSON フルバックアップ / 自動保存

## 主な機能

### 商品管理
| 機能 | 説明 |
|------|------|
| 商品マスター | 商品名・品番・カテゴリ・JANコード・内容量・賞味期限・保存方法 |
| 完成度スコア | 入力必須項目の充足率を自動計算 |
| フィルター | すべて / スター / 公開中 / 下書き / 未完成 / 賞味期限未設定 等 |
| 検索 | 商品名・品番・カテゴリでリアルタイム検索 |
| 削除 undo | 削除後 5 秒間「元に戻す」トーストを表示 |

### 原材料管理
| 機能 | 説明 |
|------|------|
| 原材料入力 | 名称・重量入力。Enter で次行へ自動移動 |
| 自動ソート | 重量順・添加物後列ソート |
| アレルゲン自動検出 | 28 品目（特定原材料 8 品目 + 準ずるもの 20 品目） |
| まとめて貼り付け | 改行区切りテキストを一括入力 |

### 食品表示ラベル
| 機能 | 説明 |
|------|------|
| リアルタイムプレビュー | 入力と同時にラベルを生成 |
| PDF 出力 | ブラウザ印刷（印刷オフセット調整可能） |
| 画像エクスポート | PNG ファイルとして保存 |
| サイズ設定 | プリセット + 自由入力（幅・高さ・フォントサイズ） |
| 複数商品一括印刷 | チェックボックスで選択して一括出力 |

### 商品規格書
| 機能 | 説明 |
|------|------|
| A4 規格書生成 | 商品情報・原材料・アレルゲン・保存方法・製造者を自動整形 |
| 原価情報切替 | 原価を含む / 含まない を選択して印刷 |
| 署名欄 | 担当者・承認者の署名欄付き |

### 原価管理
| 機能 | 説明 |
|------|------|
| 直接原価モード | 材料費・包装費・送料・その他を直接入力 |
| 明細モード | 原材料別に数量・価格を入力して合計自動計算 |
| KPI 自動計算 | 原価率・粗利率・粗利額 |
| ダッシュボード | 平均原価率・最高粗利率商品をサマリー表示 |

### AI 機能（OpenAI API 連携）
| 機能 | 説明 |
|------|------|
| 食品表示法チェック | gpt-4o-mini による法令準拠チェック・改善提案 |
| AI 商品説明文 | 楽天・Amazon・Yahoo 等チャネル別の商品説明文を自動生成 |
| AI 相談チャット | 商品ごとのコンテキスト付きチャット（専門家への相談をシミュレート） |
| API 未登録時 | テンプレート回答にフォールバック（すべての機能が引き続き使える） |

### データ管理
| 機能 | 説明 |
|------|------|
| 自動保存 | 入力から 2 秒後に自動保存（ステータスインジケーター付き） |
| JSON バックアップ | 原材料・アレルゲン情報を含む全データをエクスポート / インポート |
| CSV エクスポート | 基本情報のみ CSV 形式で出力 |
| チーム共有 | JSON ファイルをメールや Slack で共有して他端末へインポート |
| 変更履歴 | 商品ごとにスナップショット保存・任意タイミングで復元 |

## ディレクトリ構成

```
/
├── index.html              # エントリーポイント
├── src/
│   ├── app.js              # メインアプリケーション（UI 生成・ビジネスロジック）
│   ├── delegation.js       # イベントデリゲーション（クリック・キーボード・input）
│   ├── state.js            # グローバル状態変数（唯一の状態管理場所）
│   ├── constants.js        # 定数（アレルゲン一覧・カテゴリ・サイズプリセット等）
│   ├── db.js               # 栄養成分 DB・原材料データ
│   ├── utils-core.js       # ユーティリティ関数（escapeHtml・uid・safeGet 等）
│   └── styles.css          # スタイルシート
├── assets/
│   └── app-icon.svg        # アプリアイコン
├── tests/                  # Playwright テスト
├── ROADMAP.md              # 開発計画
├── TODO.md                 # 改善タスク管理
├── CHANGELOG.md            # 変更履歴
├── PROJECT_RULES.md        # 開発ルール
└── README.md               # このファイル
```

### スクリプトロード順序（重要）

```
db.js → constants.js → utils-core.js → state.js → delegation.js → app.js
```

この順序を変更するとアプリが起動しません。

## 開発方法

### ローカルで起動する

```bash
# 任意のローカルサーバーで index.html を配信する
# 例: Python
python -m http.server 3001

# 例: Node.js (serve)
npx serve . -p 3001
```

### 変更後のバージョン更新

`index.html` のバージョン文字列を更新してブラウザキャッシュをクリアする:

```html
<!-- 変更前 -->
<link rel="stylesheet" href="./src/styles.css?v=20260709m" />
<!-- 変更後（末尾のアルファベットをインクリメント） -->
<link rel="stylesheet" href="./src/styles.css?v=20260709n" />
```

`index.html` 内の `?v=` は全ファイル（CSS・JS 5本）で統一する。

### テスト実行

```bash
npm test
```
